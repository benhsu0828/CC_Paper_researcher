這是無人值守的批次任務，沒有人類可回答問題。所有 skill 中途的決策點，一律採用下列預設值，**不要停下來詢問**：

1. **寫作風格固定為 `academic`（學術深度解讀）**，跳過風格選擇步驟，直接進入產出。
2. 全文使用**台灣繁體中文**（HTML 與 JSON 的值都必須是繁體字，**嚴禁簡體字**，例如要寫「網路/實驗/動作/識別」而非「网络/实验/动作/识别」）。
3. HTML 報告寫到：`{out_dir}/report.html`（自包含）。
4. **報告中務必包含一張「方法流程圖」**（用 `<pre class="mermaid">` 或 ```mermaid 區塊，放在「快速抓重點 → 方法流程」子段，讀者最想看的部分）。先把整篇論文的 pipeline 整理清楚「再」畫，不要隨便畫：
   - 畫成**分層的 Input → 模型各階段 → Output**（`flowchart TB` + 每階段一個 `subgraph`：輸入層／特徵萃取／融合或核心模組／解碼或預測／輸出層，依論文調整）。
   - **每個節點寫出具體模組真名 ＋ 它的產出/作用**（如 VideoMAE、Token Refiner、Cross-Attention），別只寫「編碼器/模型」；用邊標籤描述流動（`➔ 產出：…`、Q/K/V、top-k）。
   - 圖要涵蓋整條實驗鏈路：原始資料輸入＋（若有）預抽取的外部特徵 → 模型各階段 → 最終輸出。
   - **圖後緊接一段逐步文字說明**：依輸入→各階段→輸出的順序，逐步講「每一步吃什麼、用什麼模組、吐出什麼、這個產出拿來幹嘛」，讓我看完就懂整篇在做什麼。
5. 公式照常用 `$$...$$`（系統會在後處理自動轉成 MathML，你不需處理）。
6. 若需要抓取全文或程式碼，只使用 arXiv 與 GitHub（其餘網域會被擋）。

**這份報告要服務「快速抓重點」的閱讀流程——前置重點、精簡後段、別想太久也別灌字數。** 務必照 academic 風格的 h2 順序產出，h2 標題需包含下列關鍵字（後處理會依關鍵字抽進 Notion）：

- **快速抓重點（What / Why / How）**：置頂，含 4 個 h3（核心問題與價值Gap／方法流程Input→Process→Output加流程圖加圖後逐步說明／資料集製作方法與來源／關鍵創新點Novelty）。
- **對自身研究的幫助**：依「上方我的研究脈絡」書寫，用「**濃縮標題：**細節」1–3 條，講清這篇對我目前進度/卡關的可用性與啟發（若上方未提供脈絡，退為對相關研究方向的通用可遷移性，不要硬掰）。
- **實驗分析**：h2 標題務必含「實驗」二字（後處理會抽進 Notion）。重點是**解讀作者的實驗數據**——主結果每個關鍵數字代表什麼、贏 baseline 多少在實務上意味著什麼；消融實驗中**哪個模組/模態貢獻最大、為什麼**。不要只貼數字。
- **討論／侷限分析／研究脈絡與前置工作／產品落地分析**：每段第一句用 `<strong>` 粗體濃縮標題講完主旨，再接細節。其中「研究脈絡與前置工作」精簡即可，重點是幫我補充該領域其他研究的最新進度／同期工作（追上 related work）。

另外：報告中每個重要主張/數字都要能在論文原文（章節／Figure／Table）找到對應，產出時自己多做確認、**嚴禁捏造**；但**不要**在 HTML 裡額外標出處（維持既有 Fig/Table/Section 編號引用即可）。
