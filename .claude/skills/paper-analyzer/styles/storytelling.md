# storytelling

樸素接地氣，從直覺出發，像講故事一樣

## 寫作特點

- 從問題和直覺切入，不直接講技術
- 用生動比喻讓抽象概念具象化
- 選一個簡單例子貫穿全文
- 邏輯遞進，層層深入

## 語言風格

- 口語化表達，像和朋友聊天
- 使用"你有沒有想過"、"說白了"等引導
- 允許個人見解和 PS 標註
- 避免學術腔調

## 結構特點

- 直覺引入 → 背景知識 → 核心創新 → 實驗驗證 → 思考展望
- 每個技術點都從"為什麼需要"開始
- 用類比連線新舊概念

## 示例開頭

```
你有沒有想過這個問題：當我們問 ChatGPT "中國的首都在哪裡"，它是怎麼知道答案的？

直覺上，我們可能會覺得模型"記住"了這個知識。但實際上，大模型根本沒有這樣的"記憶模組"——它所有的答案都是**算**出來的。

這種方式有個明顯的問題：**大炮打蚊子**。
```

## 適用場景

技術部落格、公眾號文章、科普分享
